Обычная очередь

Очередь работает по принципу FIFO (First In — First Out): кто первым встал в очередь, тот первым выйдет.

Очередь в магазине:

👤 Анна
прибыла 9:00
👤 Борис
прибыл 9:05
👤 Вера
прибыла 9:10
← обслуживается первой

Анна пришла первой → её обслуживают первой. Всё справедливо!

from collections import deque

queue = deque()
# Добавить в конец
queue.append("Анна")   # [Анна]
queue.append("Борис")  # [Анна, Борис]
queue.append("Вера")   # [Анна, Борис, Вера]

# Забрать из начала
first = queue.popleft()  # Анна
print(first)  # Анна
# Очередь: [Борис, Вера]

Проблема обычной очереди

Что если у пациентов в больнице разная срочность? Или у задач в ОС разный приоритет?

Очередь FIFO не учитывает приоритет — она просто берёт тех, кто пришёл раньше.

⚠ «Скорая помощь» должна объехать всех, кто позвонил раньше, даже если впереди стоит человек с простудой!

Очередь с приоритетом

Очередь с приоритетом всегда выдаёт элемент с наименьшим (или наибольшим) приоритетом, независимо от порядка добавления.

Порядок обслуживания:

ПациентВремяПриоритет
Анна9:003 (низкий)
Борис9:051 (критично)
Вера9:102 (средний)

Обслуживаются: Борис → Вера → Анна

Как это устроено внутри?

Используется специальная структура данных — бинарная куча (Binary Heap). Это дерево, в котором каждый родитель меньше своих детей.

heapq

Модуль heapq в Python

heapq реализует min-кучу (min-heap): heappop() всегда извлекает наименьший элемент за O(log n).

import heapq

# Создаём кучу из списка
heap = [5, 3, 8, 1, 4]
heapq.heapify(heap)
print(heap)  # [1, 3, 8, 5, 4] (куча)

# Добавить элемент
heapq.heappush(heap, 2)
# heap теперь содержит 2 на правильном месте

# Извлечь минимум
min_val = heapq.heappop(heap)
print(min_val)  # 1

# Посмотреть минимум без извлечения
print(heap[0])  # текущий минимум
heapify(lst)
Превратить список в кучу за O(n). Изменяет список на месте.
heappush(h, x)
Добавить x в кучу за O(log n). Куча остаётся упорядоченной.
heappop(h)
Извлечь и вернуть минимальный элемент за O(log n).
h[0]
Посмотреть минимум без удаления за O(1).
heapq

heapq с кортежами — мощный приём

Если в кучу класть кортежи (приоритет, данные), то Python автоматически сортирует их по первому элементу (приоритету).

import heapq

# Очередь задач с приоритетом
tasks = []

# Добавляем (приоритет, название)
heapq.heappush(tasks, (3, "Написать отчёт"))
heapq.heappush(tasks, (1, "Починить сервер"))
heapq.heappush(tasks, (2, "Ответить на письма"))

# Извлекаем в порядке приоритета
while tasks:
    priority, name = heapq.heappop(tasks)
    print(f"[{priority}] {name}")

# [1] Починить сервер
# [2] Ответить на письма
# [3] Написать отчёт
import heapq

# Для max-heap: храним отрицательные числа
# (heapq — только min-heap!)

nums = [3, 1, 4, 1, 5, 9]
max_heap = []

for n in nums:
    heapq.heappush(max_heap, -n)  # минус!

max_val = -heapq.heappop(max_heap)
print(max_val)  # 9 (наибольший)

# Для "тупиков" (путей):
# (время_отправления, номер_тупика)
# → при равном времени: меньший номер
occupied = []
heapq.heappush(occupied, (15, 3))
heapq.heappush(occupied, (12, 1))
print(occupied[0])  # (12, 1)
Трюк с max-heap: heapq всегда min-куча. Чтобы получить max-кучу, храни отрицательные числа: heappush(h, -x), max = -heappop(h).
Жадные алгоритмы

Жадный алгоритм

Жадный алгоритм на каждом шаге выбирает локально оптимальное решение, надеясь, что это приведёт к глобально оптимальному результату.

🏃

Принцип

На каждом шаге делаем лучший из доступных выборов, не пересматривая прошлые решения.

✅

Когда работает

Когда задача обладает «жадным свойством»: локально оптимальный выбор никогда не хуже любого другого.

📌

Примеры

Сдача монетами, расписание задач, алгоритм Дейкстры, сжатие Хаффмана.

В нашей задаче: Жадный выбор — всегда брать свободный тупик с наименьшим номером. Это гарантирует оптимальное распределение, потому что нам не важно какой конкретно тупик занять — главное что он свободен.
Жадные алгоритмы

Почему задача о поездах — жадная?

Поезда отсортированы по времени прибытия. Для каждого поезда нам нужно найти свободный тупик. Жадный выбор — взять тупик с наименьшим номером.

Стратегия на каждом шаге:

1️⃣
Освободить тупики, поезда из которых уже уехали до прибытия нового поезда (departed < arrived).
2️⃣
Взять свободный тупик с наименьшим номером (жадный выбор).
3️⃣
Поместить тупик в "занятые" с временем отправления нового поезда.
4️⃣
Если свободных нет — ответить: «невозможно» и вывести, сколько поездов успели разместить.
Задача V3

Задача: Распределение поездов по тупикам

E. Тупики

На вокзале K тупиков. Когда электричка прибывает, её ставят в свободный тупик с минимальным номером. Тупик освобождается после отправления электрички (если уехала в момент X, то с момента X+1 тупик свободен).

По расписанию (время прибытия и отправления, упорядочены по прибытию) определите, в какой тупик прибудет каждая электричка.

Формат ввода

Числа K и N (1 ≤ K, N ≤ 100000). Далее N строк: время прибытия и отправления каждой электрички.

Формат вывода

N номеров тупиков. Если тупиков не хватит — «0 номер_первой_проблемной_электрички».

Ввод
1 1
1 2
5
Вывод
1
Две кучи:
  • free — min-куча свободных тупиков (по номеру)
  • occupied — min-куча занятых тупиков (по времени отправления)
import heapq

k, n = map(int, input().split())
trains = []
for _ in range(n):
    arr, dep = map(int, input().split())
    trains.append((arr, dep))

# Свободные тупики: 1, 2, ..., k
free = list(range(1, k + 1))
heapq.heapify(free)

# Занятые: (время_отправления, номер_тупика)
occupied = []
result = []

for arr, dep in trains:
    # Освобождаем тупики, поезда ушли до arr
    while occupied and occupied[0][0] < arr:
        _, track = heapq.heappop(occupied)
        heapq.heappush(free, track)

    if free:
        track = heapq.heappop(free)
        heapq.heappush(occupied, (dep, track))
        result.append(track)
    else:
        print(0, len(result) + 1)
        exit()

for r in result:
    print(r)

Что мы изучили?

heapq

  • Min-куча: pop = минимум
  • heappush(h, x)
  • heappop(h)
  • heapify(lst)
  • Кортеж = (приоритет, данные)

Жадный алгоритм

  • Локально лучший выбор
  • Не пересматривает прошлые
  • Работает когда есть «жадное свойство»

Задача о поездах

  • Две кучи: free и occupied
  • Освобождаем ушедшие тупики
  • Берём наименьший номер
  • Жадно → оптимально
Главный инсайт: heapq позволяет эффективно находить минимум из динамически изменяющегося набора — O(log n) на каждую операцию. Это идеальный инструмент для жадных алгоритмов, где на каждом шаге нужен «лучший» свободный ресурс.
