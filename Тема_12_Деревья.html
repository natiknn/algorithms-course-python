Теория — BST

Дерево двоичного поиска (BST)

Свойство BST

Для каждого узла: все значения в левом поддереве < узел < все значения в правом поддереве.

8
3 10
1 6 14

Дерево: 1 < 3 < 6 < 8 < 10 < 14

Операции BST

class BSTNode:
    def __init__(self, val):
        self.val = val
        self.left = None
        self.right = None

def bst_search(root, target):
    if root is None or root.val == target:
        return root
    if target < root.val:
        return bst_search(root.left, target)
    return bst_search(root.right, target)

def bst_insert(root, val):
    if root is None:
        return BSTNode(val)
    if val < root.val:
        root.left = bst_insert(root.left, val)
    else:
        root.right = bst_insert(root.right, val)
    return root
✅ Inorder-обход BST даёт отсортированную последовательность!
Теория — Проблема несбалансированности

Зачем нужен баланс?

❌ Несбалансированное BST — O(n)

Вставляем числа 1, 2, 3, 4, 5 по порядку:

1
└── 2
└── 3
└── 4
└── 5
Дерево превращается в связный список! Высота = n. Поиск = O(n).

✅ Сбалансированное BST — O(log n)

Те же числа 1, 2, 3, 4, 5 в сбалансированном дереве:

3
1 4
2 5
Высота = log₂(n). Поиск, вставка, удаление = O(log n).
💡 Сравнение сложностей:
Несбал.: поиск O(n), вставка O(n)
Сбал.: поиск O(log n), вставка O(log n)
Теория — AVL-дерево

AVL-дерево: автоматический баланс

Инвариант AVL

Для каждого узла: |высота(левого) - высота(правого)| <= 1.

Коэффициент баланса (bf) = высота(right) - высота(left)

  • bf = -1, 0, 1 — узел сбалансирован
  • bf = -2 — перевес влево → вращение вправо
  • bf = +2 — перевес вправо → вращение влево

Типы вращений

L
Левое вращение — при bf = +2
R
Правое вращение — при bf = -2
LR
Лево-правое — при bf = -2, bf(left) = +1
RL
Право-левое — при bf = +2, bf(right) = -1

Пример: правое вращение

Вставили 5, 3, 1 — дерево разбалансировано (bf=-2 у корня 5):

5 (bf=-2) ← несбалансировано!
└── 3 (bf=-1)
└── 1

После правого вращения (новый корень = 3):

3
1 5
✅ Баланс восстановлен! bf у всех узлов = 0.
💡 AVL используется в C++ STL (map, set). Строго сбалансирован — поиск чуть быстрее, вставка чуть медленнее, чем у Red-Black.
Теория — AVL-дерево

Что такое высота узла?

Определение

Высота узла — это количество рёбер на самом длинном пути от этого узла вниз до листа.

∅
Пустой узел (None) → высота = −1
0
Лист (нет детей) → высота = 0
+1
Каждый уровень выше добавляет +1
💡 Почему None = −1, а не 0? Чтобы формула bf работала правильно: лист имеет bf = (−1) − (−1) = 0.

Пример вычисления высот

5 ← высота 2
└── 3 ← высота 1
├── 1 ← высота 0
└── 4 ← высота 0
Узел 3: путь до листа = 1 ребро → высота = 1
Узел 5: путь до листа = 2 ребра → высота = 2

Как считается bf у узла 5?

bf = высота(right) − высота(left) = −1 − 1 = −2

Правого поддерева нет (None → −1), левое = высота 1. bf = −2 → несбалансировано!

Теория — AVL-дерево

Правое и левое вращение

Правое вращение (bf = −2, перевес влево)

Дерево "падает влево" — поднимаем левого ребёнка вверх:

5bf=−2
└──3
└──1
→
3новый корень
├──1
└──5
✅ Узел 3 поднялся, узел 5 опустился вправо. Порядок 1 < 3 < 5 сохранился!

Левое вращение (bf = +2, перевес вправо)

Зеркальная ситуация — поднимаем правого ребёнка:

1bf=+2
└──3
└──5
→
3новый корень
├──1
└──5
✅ Узел 3 поднялся, узел 1 опустился влево. Порядок 1 < 3 < 5 сохранился!
🔑 Главное правило:
Перевес влево (bf=−2) → вращаем вправо (поднимаем левого ребёнка)
Перевес вправо (bf=+2) → вращаем влево (поднимаем правого ребёнка)
Теория — Red-Black Tree

Красно-чёрное дерево (Red-Black)

Свойства Red-Black дерева

1
Каждый узел — красный или чёрный
2
Корень всегда чёрный
3
Красный узел не может иметь красного потомка (нет двух красных подряд)
4
От любого узла до листа (null) одинаковое количество чёрных узлов
💡 Эти свойства гарантируют: высота <= 2·log₂(n+1), то есть O(log n).

Теория — Red-Black Tree

Красно-чёрное дерево: как это работает

Зачем вообще два цвета?

Цвет — это просто метка (один бит: красный/чёрный). Сами по себе цвета ничего не значат. Но правила для этих цветов косвенно не дают дереву стать слишком высоким.

💡 AVL следит за высотой напрямую.
Red-Black следит за цветами — и это косвенно контролирует высоту. Меньше работы при вставке.

Как правило №4 гарантирует баланс?

Правило: от любого узла до листа одинаковое количество чёрных узлов (чёрная высота).

Допустим, чёрная высота = 2:

↓
Кратчайший путь: чёрный → чёрный = 2 узла
↓
Длиннейший путь: чёрный → красный → чёрный → красный = 4 узла
✅ Длиннейшая ветка максимум вдвое длиннее кратчайшей.
Поэтому высота <= 2·log₂(n+1) → поиск O(log n) гарантирован.

Пример валидного Red-Black дерева

Числа 1, 2, 3, 4, 5 в Red-Black дереве:

2 ← чёрный (корень)
1 чёрный
4 красный
3 чёрный
5 чёрный
Проверка правила №4: от корня (2) до любого листа — ровно 2 чёрных узла.
Правило №3: красный (4) → дети чёрные (3 и 5). ✅
Теория — Red-Black Tree

Red-Black vs AVL: что выбрать?

Главное отличие в подходе

AVL
Строгий баланс — разница высот <= 1. После каждой вставки может потребоваться несколько вращений, чтобы восстановить баланс.
R-B
Нестрогий баланс — дерево может быть чуть выше, но вставка требует максимум 2 вращения. Остальное — перекраска узлов.
🔑 Перекраска = просто изменить бит в памяти. Это намного быстрее вращения (перестановки указателей).

Аналогия

AVL — перфекционист: после каждого изменения выравнивает всё идеально.
Red-Black — прагматик: делает минимум работы, лишь бы не стать совсем кривым.

Когда что использовать

СитуацияВыборПочему
Много чтений, мало записейAVLДерево ниже → поиск быстрее
Много вставок/удаленийRed-BlackМеньше вращений при изменении
Общий случай (базы, ОС)Red-BlackЛучший баланс скоростей

Где используется в реальном мире

TreeMap, TreeSet — Red-Black

std::map, std::set — Red-Black

Планировщик задач CFS — Red-Black
✅ В реальных системах вставок и удалений больше, чем чистых поисков — поэтому Red-Black победил.
Python — sortedcontainers

SortedList: сбалансированное дерево в Python

В стандартной библиотеке Python нет BST. Но есть sortedcontainers — эффективная реализация на чистом Python.

from sortedcontainers import SortedList

sl = SortedList()

# Добавление — O(log n)
sl.add(5)
sl.add(1)
sl.add(3)
sl.add(7)
sl.add(2)
# sl = SortedList([1, 2, 3, 5, 7])

# Удаление — O(log n)
sl.remove(3)       # удалить 3
sl.discard(99)     # ничего если нет

# Поиск — O(log n) через bisect
sl.bisect_left(5)  # → 2  (индекс вставки)
sl.bisect_right(5) # → 3

# Доступ по индексу — O(log n)
sl[0]   # → 1 (минимум)
sl[-1]  # → 7 (максимум)
sl[1]   # → 2
# Проверка принадлежности — O(log n)
5 in sl   # True
4 in sl   # False

# Количество элементов в диапазоне — O(log n)
sl.count(5)             # 1
sl.irange(2, 6)         # итератор [2, 5]
# список элементов от 2 до 6 включительно

# k-й элемент по порядку — O(log n)
sl[2]   # 3-й наименьший

# SortedList с дубликатами
sl2 = SortedList([3, 1, 4, 1, 5, 9, 2, 6])
# SortedList([1, 1, 2, 3, 4, 5, 6, 9])
sl2.count(1)   # 2

from sortedcontainers import SortedList

class KthSmallest:
    def __init__(self):
        self.sl = SortedList()

    def add(self, num):
        self.sl.add(num)

    def kth(self, k):
        return self.sl[k - 1]  # O(log n)

# SortedList хранит все, доступ по индексу

2. Количество элементов в диапазоне

sl = SortedList([1, 3, 5, 7, 9, 11])

# Сколько чисел от 3 до 9?
lo = sl.bisect_left(3)   # 1
hi = sl.bisect_right(9)  # 5
count = hi - lo          # 4  ← [3,5,7,9]

# Или через irange
len(list(sl.irange(3, 9)))  # 4

3. Медиана в скользящем окне размером k

from sortedcontainers import SortedList

def medianSlidingWindow(nums, k):
    window = SortedList(nums[:k])
    result = []

    def get_median():
        if k % 2 == 1:
            return float(window[k // 2])
        return (window[k//2 - 1] + window[k//2]) / 2

    result.append(get_median())

    for i in range(k, len(nums)):
        window.add(nums[i])          # O(log k)
        window.remove(nums[i - k])   # O(log k)
        result.append(get_median())  # O(log k)

    return result

Count of Smaller Numbers After Self

Условие

Для каждого элемента nums[i] найти количество чисел правее него, которые меньше nums[i].

Вход: [5, 2, 6, 1]
Выход: [2, 1, 1, 0]
5: правее меньше 5 → {2,1} → 2
2: правее меньше 2 → {1} → 1

Алгоритм через SortedList

1
Обходим массив справа налево
2
Добавляем каждый элемент в SortedList
3
bisect_left(nums[i]) = количество элементов в SortedList, которые строго меньше nums[i]
from sortedcontainers import SortedList

def countSmaller(nums):
    sl = SortedList()
    result = []

    for num in reversed(nums):
        # bisect_left = индекс вставки = кол-во меньших
        smaller_count = sl.bisect_left(num)
        result.append(smaller_count)
        sl.add(num)

    return result[::-1]  # реверс т.к. обходили справа

# Трассировка: [5, 2, 6, 1]
# num=1: sl=[], bisect_left(1)=0 → [0]; sl=[1]
# num=6: sl=[1], bisect_left(6)=1 → [0,1]; sl=[1,6]
# num=2: sl=[1,6], bisect_left(2)=1 → [0,1,1]; sl=[1,2,6]
# num=5: sl=[1,2,6], bisect_left(5)=2 → [0,1,1,2]
# Реверс → [2, 1, 1, 0] ✓

O(n log n)
Память

E. Тупики — SortedList для минимального свободного слота

Условие (кратко)

На вокзале K тупиков (1..K). Прибывают N электричек (отсортированы по времени прибытия). Каждую электричку ставят в свободный тупик с минимальным номером. Если тупик освободился в момент X (отправление), электричка, прибывающая в тот же момент X, использовать его не может.

Вход: K=2, N=3
пары (прибытие, отправление): (1,5), (2,4), (6,10)
Выход: 1, 2, 1

Алгоритм — два инструмента

1
SortedList(1..K) — множество свободных тупиков. sl[0] = минимальный.
2
heapq — занятые тупики: (время_отправления, номер). Вершина = ближайшее освобождение.
3
Для каждой прибывающей электрички (время a): сначала освободить все тупики с отправлением строго меньше a, затем взять sl[0].
💡 Условие «строго меньше»: если отправление = X и прибытие = X, тупик ещё занят. Прибытие X+1 — уже свободен. 
import heapq
from sortedcontainers import SortedList

K, N = map(int, input().split())

free = SortedList(range(1, K + 1))  # свободные тупики
occupied = []   # min-heap (время_отпр, тупик)
result = []
failed = -1

for i in range(N):
    a, d = map(int, input().split())

    # Освободить тупики с отправлением строго < a
    while occupied and occupied[0][0] < a:
        dep_time, track = heapq.heappop(occupied)
        free.add(track)          # O(log K)

    if not free:
        failed = i + 1           # 1-based номер электрички
        break

    track = free[0]              # минимальный свободный
    free.remove(track)           # O(log K)
    heapq.heappush(occupied, (d, track))
    result.append(track)

if failed != -1:
    print(0, failed)
else:
    for r in result:
        print(r)

E. Менеджер памяти — интервалы на SortedList

Условие (кратко)

Память — отрезки 1..N. K запросов двух типов:

  • 1 l r — объединить отрезки l..r в группу, если ни один из них уже не в группе. Вывести 1 (успех) или 0 (занято).
  • 2 d — удалить группу, содержащую отрезок d. Вывести первый и последний отрезок группы, или «0 0» если d не в группе.
Пример: 5 отрезков, 6 запросов:
1 1 2 → 1 (сгруппировали [1,2])
1 4 5 → 1 (сгруппировали [4,5])
1 2 4 → 0 (отрезок 2 занят!)
2 5 → 4 5 (удалили группу [4,5])
1 2 → 1 2 (удалили группу [1,2])
2 4 → 0 0 (отрезок 4 не в группе)
💡 Ключевая операция: проверить, пересекается ли новый интервал [l,r] с уже существующими группами. bisect_right(r) даёт позицию для поиска ближайшего кандидата справа. 
from sortedcontainers import SortedList

N, K = map(int, input().split())

# Храним начала групп в SortedList
starts = SortedList()
# starts[i] → конец группы
ends = {}   # start → end

def has_overlap(l, r):
    """Проверить, пересекается ли [l,r] с любой группой"""
    # Ближайшая группа, начинающаяся <= r
    idx = starts.bisect_right(r) - 1
    if idx >= 0:
        gs = starts[idx]
        ge = ends[gs]
        if ge >= l:   # конец группы >= l → пересечение
            return True
    return False

for _ in range(K):
    q = list(map(int, input().split()))
    if q[0] == 1:
        l, r = q[1], q[2]
        if has_overlap(l, r):
            print(0)
        else:
            starts.add(l)    # O(log n)
            ends[l] = r
            print(1)
    else:
        d = q[1]
        # Найти группу, содержащую d
        idx = starts.bisect_right(d) - 1
        if idx >= 0:
            gs = starts[idx]
            ge = ends[gs]
            if gs <= d <= ge:
                print(gs, ge)
                starts.remove(gs)   # O(log n)
                del ends[gs]
                continue
        print(0, 0)
