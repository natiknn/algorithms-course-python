Ключевое наблюдение

Смотрим на крайние символы подстроки s[i..j]:

Случай 1: s[i] == s[j]

Символы уже «закрывают» друг друга. Задача сводится к решению внутреннего отрезка s[i+1..j-1].

Случай 2: s[i] != s[j]

Нужно добавить одну вставку. Либо «закрываем» левый символ (вставляем s[i] справа), либо правый (вставляем s[j] слева). Берём минимум из двух вариантов.

💡

Это классический Interval DP — решаем задачу на всё более длинных подстроках.

// если s[i] == s[j]
dp[i][j] = dp[i+1][j-1]

// иначе
dp[i][j] = 1 + min(
  dp[i+1][j],
  dp[i][j-1]
)

// база: dp[i][i] = 0

Что хранится в dp[i][j]?

dp[i][j]

Минимальное количество вставок, необходимых чтобы превратить подстроку s[i..j] в палиндром.

База (длина = 1)

Строка из одного символа — уже палиндром.

dp[i][i] = 0 для всех i

Ответ

После заполнения всей таблицы, ответ лежит в:

dp[0][n-1] // вся строка

Порядок заполнения: по возрастанию длины подстроки

Python 
# длина 1 → длина 2 → ... → длина n
for length in range(2, n + 1):
    for i in range(n - length + 1):
        j = i + length - 1
        # dp[i][j] всегда знает dp[i+1][j-1]
        # т.к. она короче и уже вычислена
✅

Заполняем только верхний треугольник матрицы (i <= j). Нижний треугольник не используется.

Interval DP — Python

Инициализация

Создаём таблицу n×n, заполненную нулями. База — диагональ (длина 1 = 0 вставок).

Внешний цикл

Перебираем длину подстроки от 2 до n. Так гарантируем, что при вычислении dp[i][j] уже известны все более короткие подстроки.

Рекуррентность

Если крайние символы совпали — берём внутренний отрезок. Иначе — 1 + min двух вариантов.

Python 
def minInsertions(s: str) -> int:
    n = len(s)

    # dp[i][j] = мин. вставок для s[i..j]
    dp = [[0] * n for _ in range(n)]

    # заполняем по возрастанию длины
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1

            if s[i] == s[j]:
                dp[i][j] = dp[i+1][j-1]
            else:
                dp[i][j] = 1 + min(
                    dp[i+1][j],
                    dp[i][j-1]
                )

    return dp[0][n-1]
⏱ O(n²) время 💾 O(n²) память

Через LCS и LPS

Формула

ответ = n − LPS(s)
LPS(s) = LCS(s, reverse(s))

Почему это работает?

Символы, входящие в наидлиннейшую палиндромную подпоследовательность (LPS), уже стоят правильно — их трогать не нужно.

Для каждого из оставшихся n − LPS символов нужна ровно одна вставка-пара.

Пример: "mbadm"

reverse = "mdabm"
LCS = "mam" → длина 3
ответ = 5 − 3 = 2 ✓
Python · LCS подход 
def minInsertions(s: str) -> int:
    n = len(s)
    t = s[::-1]  # реверс строки

    # dp[i][j] = LCS(s[:i], t[:j])
    dp = [[0] * (n + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        for j in range(1, n + 1):
            if s[i-1] == t[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(
                    dp[i-1][j],
                    dp[i][j-1]
                )

    lps = dp[n][n]
    return n - lps

📝 Задача 3: Код Сложная

Напиши функцию, которая возвращает сам палиндром (а не только количество вставок) для строки "race".

После заполнения dp[i][j] восстанавливай ответ рекурсивно: если s[i]==s[j] — идём в dp[i+1][j-1] и оборачиваем; иначе идём в меньшее из dp[i+1][j] или dp[i][j-1].
Python
def buildPalindrome(s, i, j, dp):
    if i > j: return ""
    if i == j: return s[i]
    if s[i] == s[j]:
        mid = buildPalindrome(s,i+1,j-1,dp)
        return s[i] + mid + s[j]
    if dp[i+1][j] < dp[i][j-1]:
        mid = buildPalindrome(s,i+1,j,dp)
        return s[i] + mid + s[i]
    mid = buildPalindrome(s,i,j-1,dp)
    return s[j] + mid + s[j]
# "race" → "ecarace"
