Рекурсия

Что такое рекурсия?

Рекурсия — это когда функция вызывает саму себя. Звучит странно, но это мощный инструмент!

# Факториал: n! = n × (n-1)!
# 5! = 5 × 4! = 5 × 4 × 3! = ...

def factorial(n):
    if n == 0:       # базовый случай
        return 1
    return n * factorial(n - 1)  # рекурсия

print(factorial(5))  # 120
# 5 × 4 × 3 × 2 × 1 = 120
Аналогия: Матрёшка! Чтобы достать последнюю куклу, открываешь текущую, а внутри снова «та же задача» — открыть следующую.

Трассировка factorial(4)

factorial(4) → 4 × factorial(3)
factorial(3) → 3 × factorial(2)
factorial(2) → 2 × factorial(1)
factorial(1) → 1 × factorial(0)
factorial(0) → 1 ✓
← 1×1×2×3×4 = 24

Базовый случай — это критически важно!

Каждая рекурсивная функция должна иметь базовый случай — условие, при котором она останавливается и возвращает результат напрямую, без нового вызова себя.

✓ С базовым случаем

def count_down(n):
    if n == 0:          # базовый случай
        print("Пуск!")
        return
    print(n)
    count_down(n - 1)   # рекурсия

count_down(3)
# 3
# 2
# 1
# Пуск!

✗ Без базового случая

def infinite(n):
    print(n)
    infinite(n - 1)  # НИКОГДА не стопнется!

# infinite(5)
# → Ошибка: RecursionError:
#   maximum recursion depth exceeded
⚠ Без базового случая программа уйдёт в бесконечную рекурсию и упадёт с ошибкой!
Правило: При написании рекурсивной функции всегда начинай с базового случая. Только убедившись, что функция умеет остановиться, добавляй рекурсивный вызов.

Рекурсивный случай

Рекурсивный случай — это когда задача слишком большая для прямого решения. Тогда мы разбиваем её на меньшую версию той же задачи и вызываем себя.

def sum_list(lst):
    # Базовый случай: пустой список
    if len(lst) == 0:
        return 0
    # Рекурсивный случай:
    # сумма = первый элемент + сумма остальных
    return lst[0] + sum_list(lst[1:])

print(sum_list([1, 2, 3, 4]))
# 1 + sum([2, 3, 4])
# 1 + 2 + sum([3, 4])
# 1 + 2 + 3 + sum([4])
# 1 + 2 + 3 + 4 + sum([])
# 1 + 2 + 3 + 4 + 0 = 10

Структура рекурсивной функции

def solve(problem):
    # 1. Базовый случай:
    #    задача уже решена без рекурсии
    if problem is small enough:
        return direct_answer

    # 2. Рекурсивный случай:
    #    разбиваем на меньшую задачу
    smaller = make_smaller(problem)
    result = solve(smaller)  # вызов себя

    # 3. Собираем ответ из результата
    return combine(result)
Важно: Каждый рекурсивный вызов должен получать меньшую версию задачи. Иначе мы никогда не придём к базовому случаю!

Стек вызовов

Когда функция вызывает саму себя, Python помнит все «незаконченные» вызовы в стеке. Они достаются обратно в порядке LIFO (последний пришёл — первый ушёл).

def f(n):
    print(f"→ f({n})")
    if n == 0:
        print(f"← f({n}) = 0")
        return 0
    result = n + f(n - 1)
    print(f"← f({n}) = {result}")
    return result

f(3)
# → f(3)
# → f(2)
# → f(1)
# → f(0)
# ← f(0) = 0
# ← f(1) = 1
# ← f(2) = 3
# ← f(3) = 6

Стек в памяти

f(3): ждёт f(2)
f(2): ждёт f(1)
f(1): ждёт f(0)
f(0): возвращает 0 ✓

Возврат идёт снизу вверх: 0 → 1 → 3 → 6

Divide & Conquer

Принцип «Разделяй и властвуй»

Divide & Conquer — стратегия решения задач в три шага:

✂️

1. Divide

Разбить задачу на несколько независимых подзадач меньшего размера

⚙️

2. Conquer

Решить каждую подзадачу рекурсивно (или напрямую, если она маленькая)

🔗

3. Combine

Собрать решения подзадач в ответ для исходной задачи

Примеры: Быстрая сортировка (QuickSort), сортировка слиянием (MergeSort), бинарный поиск, умножение Карацубы — все используют Divide & Conquer.
В нашей задаче: Мы разбиваем строку по «плохим» символам — тем, которые не могут быть в ответе. Каждая часть решается независимо рекурсивно.

Разделение по «плохим» символам

«Плохой» символ — тот, который встречается в строке слишком мало раз (меньше k). Такой символ не может войти в ответ, поэтому он разделяет строку на независимые части.

Пример: s = "aababba", k = 3

Counter: a→5, b→2. Буква 'b' встречается только 2 раза < 3 → 'b' плохая!

Итоговый ответ: max(2, 1, 0, 1) = 2

Почему это работает? Плохой символ не может входить в ответную подстроку. Значит, любая допустимая подстрока целиком лежит в одной из частей. Мы решаем задачу на каждой части независимо.
Задача V2

Задача: Длиннейшая подстрока с k вхождениями каждого символа

D. Наибольшая подстрока

Дана строка s и число k. В строке s требуется найти подстроку максимальной длины, в которой все различные символы встречаются не менее k раз.

Формат ввода

Первая строка: n и k (1 <= n, k <= 2·10⁵). Вторая: строка s из n строчных латинских букв.

Формат вывода

Одно число — максимальная длина подстроки, где каждый символ встречается >= k раз.

Ввод 1
5 3
aaabb
Вывод 1
3
Ввод 2
6 2
ababbc
Вывод 2
5
Задача: Найти длину наидлиннейшей подстроки, в которой каждый символ встречается не менее k раз. 
from collections import Counter

def longest(s, k):
    # Базовый случай: строка слишком короткая
    if len(s) < k:
        return 0

    cnt = Counter(s)

    for c in cnt:
        if cnt[c] < k:
            # Нашли плохой символ — делим строку!
            parts = s.split(c)
            # Решаем задачу для каждой части
            return max(longest(p, k)
                       for p in parts)

    # Базовый случай 2:
    # Все символы встречаются >= k раз
    # Вся строка — ответ
    return len(s)

n, k = map(int, input().split())
s = input().strip()
print(longest(s, k))
Задача V2

Разбор кода по шагам

Трассировка: s = "aaabb", k = 2

longest("aaabb", 2): cnt = {a:3, b:2} — все >= 2 → return 5

Трассировка: s = "aaabbc", k = 2

longest("aaabbc", 2): cnt = {a:3, b:2, c:1} → c плохой!
  split('c') → ["aaabb", ""]
longest("aaabb", 2): cnt = {a:3, b:2} — все >= 2 → return 5
longest("", 2): len < 2 → return 0
← max(5, 0) = 5
Элегантность: Мы не думаем «как проверить все подстроки». Мы просто говорим: «плохие символы не могут быть в ответе — разбиваем по ним». Это суть Divide & Conquer!
