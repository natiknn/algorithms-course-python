Что такое порядковая статистика?

k-я порядковая статистика массива — это k-й по величине элемент в отсортированном виде.

Примеры

Массив: [3, 1, 4, 1, 5, 9, 2, 6]

  • Минимум = 1-я порядковая статистика
  • Максимум = n-я порядковая статистика
  • Медиана = ⌊n/2⌋-я статистика

Зачем нам это нужно?

  • Медиана массива — устойчивое среднее
  • Top-K рекомендации — k лучших товаров
  • Перцентили — 95-й перцентиль задержки
  • Алгоритм k-means — медиана кластера

Варианты решения

❌ Сортировка + взять k-й элемент — O(n log n)
⚡ heapq.nlargest / nsmallest — O(n log k)
🚀 QuickSelect — O(n) среднее
Теория — Подходы

Наивный O(n log n) и heapq O(n log k)

❌ Наивный — сортировка

def kth_smallest_naive(nums, k):
    return sorted(nums)[k - 1]  # O(n log n)

def kth_largest_naive(nums, k):
    return sorted(nums)[-k]     # O(n log n)
Зачем сортировать весь массив, если нас интересует один элемент?

⚡ heapq — O(n log k)

import heapq

nums = [3, 1, 4, 1, 5, 9, 2, 6]

# k наибольших
heapq.nlargest(3, nums)   # [9, 6, 5]

# k наименьших
heapq.nsmallest(3, nums)  # [1, 1, 2]

# k-й наибольший
def kth_largest(nums, k):
    return heapq.nlargest(k, nums)[-1]
# O(n log k) — лучше сортировки при малом k
При k << n это намного быстрее полной сортировки. Например, k=10, n=10⁶: log k ≈ 3 vs log n ≈ 20.

Мин-куча размером k — ручная реализация

Храним кучу из k наибольших элементов. Вершина кучи — наименьший из k наибольших = k-й наибольший.

def kth_largest_heap(nums, k):
    heap = []
    for num in nums:
        heapq.heappush(heap, num)
        if len(heap) > k:
            heapq.heappop(heap)  # выбросить минимум
    return heap[0]  # k-й наибольший

# Трассировка: nums=[3,1,4,1,5], k=2
# heap=[3]    → [1,3]
# heap=[1,3]  → [1,3,4] → pop → [3,4]
# heap=[3,4]  → [1,3,4] → pop → [3,4]
# heap=[3,4]  → [3,4,5] → pop → [4,5]
# Ответ: heap[0] = 4  ✓ (2-й наибольший)
Теория — QuickSelect

QuickSelect: поиск k-го элемента за O(n)

QuickSelect — алгоритм, похожий на QuickSort. Разница: мы рекурсивно уходим только в одну половину, а не в обе.

Идея — партиция (partition)

1
Выбрать опорный элемент (pivot)
2
Разделить массив: слева от pivot — меньше, справа — больше
3
Если pivot на позиции k — нашли! Если k меньше — идём влево. Если k больше — идём вправо.
Среднее: T(n) = T(n/2) + O(n) → O(n)
Худший случай: O(n²) (плохой pivot) 
import random

def quickselect(nums, k):
    """Найти k-й наименьший элемент (1-based)"""
    def partition(lo, hi):
        pivot_idx = random.randint(lo, hi)
        nums[pivot_idx], nums[hi] = nums[hi], nums[pivot_idx]
        pivot = nums[hi]
        store = lo
        for i in range(lo, hi):
            if nums[i] <= pivot:
                nums[i], nums[store] = nums[store], nums[i]
                store += 1
        nums[store], nums[hi] = nums[hi], nums[store]
        return store

    lo, hi = 0, len(nums) - 1
    target = k - 1  # 0-based
    while True:
        pivot_pos = partition(lo, hi)
        if pivot_pos == target:
            return nums[pivot_pos]
        elif pivot_pos < target:
            lo = pivot_pos + 1
        else:
            hi = pivot_pos - 1

O(n)
Худшее
O(n²)

O(1)

Теория — Медиана онлайн

Медиана в скользящем окне: два heap

Задача: поток чисел поступает по одному. После каждого нового числа — найти текущую медиану за O(log n).

Идея: два heap

1
max-heap (левая половина) — хранит меньшие числа. Вершина = максимум левых.
2
min-heap (правая половина) — хранит большие числа. Вершина = минимум правых.
3
Инвариант: |left| - |right| ≤ 1. Медиана = вершина большего heap или среднее двух вершин.
💡 Python не имеет max-heap. Храним отрицательные числа в min-heap: heappush(left, -x), тогда вершина = -left[0] 
import heapq

class MedianFinder:
    def __init__(self):
        self.left = []   # max-heap (отриц. числа)
        self.right = []  # min-heap

    def addNum(self, num):
        heapq.heappush(self.left, -num)

        # left.max <= right.min
        if self.right and -self.left[0] > self.right[0]:
            val = -heapq.heappop(self.left)
            heapq.heappush(self.right, val)

        # балансировка размеров
        if len(self.left) < len(self.right):
            val = heapq.heappop(self.right)
            heapq.heappush(self.left, -val)
        elif len(self.left) > len(self.right) + 1:
            val = -heapq.heappop(self.left)
            heapq.heappush(self.right, val)

    def findMedian(self):
        if len(self.left) > len(self.right):
            return -self.left[0]
        return (-self.left[0] + self.right[0]) / 2

Медиана в скользящем окне фиксированного размера

#480 · Hard Дан массив и размер окна k. Найти медиану в каждом окне.

Пример: nums=[1,3,-1,-3,5,3,6,7], k=3

ОкноЭлементыМедиана
[0..2][1,3,-1] → [-1,1,3]1
[1..3][3,-1,-3] → [-3,-1,3]-1
[2..4][-1,-3,5] → [-3,-1,5]-1
[3..5][-3,5,3] → [-3,3,5]3
[4..6][5,3,6] → [3,5,6]5
[5..7][3,6,7] → [3,6,7]6
Ответ: [1.0, -1.0, -1.0, 3.0, 5.0, 6.0]

Простой подход O(n·k) — достаточно для экзамена

def medianSlidingWindow(nums, k):
    result = []
    for i in range(len(nums) - k + 1):
        window = sorted(nums[i:i+k])
        if k % 2 == 1:
            result.append(float(window[k//2]))
        else:
            result.append(
                (window[k//2-1] + window[k//2]) / 2
            )
    return result

O(n·k log k)

  • max-heap = heappush(h, -x)
  • Медиана нечёт. = sorted[n//2]
  • Медиана чёт. = среднее двух средних
  • Рандомный pivot в QuickSelect избегает худшего случая
