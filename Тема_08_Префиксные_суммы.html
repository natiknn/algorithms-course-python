Тема 8

Префиксные суммы

Считаем сумму подмассива за O(1) вместо O(n)
— один из самых полезных трюков в алгоритмах

Prefix Sum Накопительная сумма Запросы на отрезке O(1) запрос B. Экспедиция
Мотивация

Зачем нужны префиксные суммы?

Представь: есть массив продаж за 365 дней. Тебя часто спрашивают: «сколько продали с 10 по 200 день?» Как отвечать быстро?

❌ Наивный подход — O(n) на запрос

Каждый раз суммируем элементы от l до r в цикле.

def range_sum(nums, l, r):
    total = 0
    for i in range(l, r + 1):
        total += nums[i]
    return total

# 1000 запросов × 365 дней = 365 000 операций 😱
Q запросов × N элементов = O(Q·N)
При Q=10⁵ и N=10⁵ это 10¹⁰ операций — слишком медленно!

✅ Префиксные суммы — O(n) предобработка + O(1) запрос

Один раз считаем накопленные суммы. Потом любой запрос — одно вычитание.

# Один раз предобрабатываем — O(n)
prefix = [0] * (n + 1)
for i in range(n):
    prefix[i+1] = prefix[i] + nums[i]

# Каждый запрос — O(1)!
def range_sum(l, r):
    return prefix[r+1] - prefix[l]
O(n) один раз + O(1) каждый запрос
При Q=10⁵ это всего 10⁵ + 10⁵ операций 🚀
Теория

Что такое префиксная сумма?

prefix[i] — это сумма всех элементов от начала массива до индекса i-1 включительно.

Пример: nums = [3, 1, 4, 1, 5, 9, 2]

Исходный массив:

30

Префиксный массив (со сдвигом — нулевой элемент = 0):

prefix[i] = prefix[i-1] + nums[i-1]
💡 Мы добавляем нулевой элемент = 0 в начало. Это упрощает формулу запроса и убирает граничные случаи.
sum(l, r) = prefix[r+1] - prefix[l]
Пример: сумма nums[2..4] = [4,1,5] = 10
prefix[5] - prefix[2] = 14 - 4 = 10 ✓
Теория — Почему работает

Почему prefix[r+1] - prefix[l]?

Это простая математика. Посмотрим на массив [3, 1, 4, 1, 5, 9, 2] и запрос sum(2, 4):

30
11
42
13
54
95
26

Нам нужна сумма зелёных элементов: 4 + 1 + 5 = 10

1
prefix[5] = сумма первых 5 элементов = 3+1+4+1+5 = 14
2
prefix[2] = сумма первых 2 элементов = 3+1 = 4
3
prefix[5] - prefix[2] = 14 - 4 = 10 ✓
Вычитая «лишний хвост» слева, мы получаем именно нужный нам отрезок!

Визуально:

prefix[5] = 3+1 + 4+1+5 = 14
prefix[2] = 3+1 = 4
─────────────────────
разница = 4+1+5 = 10

Общая формула:
sum(l, r) = prefix[r+1] - prefix[l]

Индексы l, r — в исходном массиве (0-based).
Индексы prefix — со сдвигом +1.
Шаблон

Шаблон — два варианта

Вариант 1: со сдвигом (рекомендую)

Массив prefix длиной n+1. Первый элемент = 0. Удобнее, нет граничных случаев.

n = len(nums)
prefix = [0] * (n + 1)

# Построение — O(n)
for i in range(n):
    prefix[i + 1] = prefix[i] + nums[i]

# Запрос sum(l, r) — O(1)
# l, r — индексы в nums (включительно)
def query(l, r):
    return prefix[r + 1] - prefix[l]

# Пример:
# nums  = [3, 1, 4, 1, 5]
# prefix= [0, 3, 4, 8, 9, 14]
# query(1, 3) = prefix[4] - prefix[1]
#             = 9 - 3 = 6  ✓  (1+4+1)
✅ Этот вариант используй по умолчанию

Вариант 2: без сдвига

Массив prefix той же длины. Нужно аккуратно обрабатывать l=0.

n = len(nums)
prefix = [0] * n

# Построение — O(n)
prefix[0] = nums[0]
for i in range(1, n):
    prefix[i] = prefix[i-1] + nums[i]

# Запрос sum(l, r) — O(1)
def query(l, r):
    if l == 0:
        return prefix[r]
    return prefix[r] - prefix[l - 1]

# ⚠️ Осторожно с l=0!
⚠️ Легко ошибиться при l=0
💡 Мнемоника для варианта 1: prefix[i] = «сколько набрали до i-го элемента (не включая его)». Поэтому сумма [l..r] = «сколько до r+1» минус «сколько до l».

#1480 · Easy

Running Sum of 1D Array ↗

Условие

Дан массив nums. Вернуть массив runningSum, где runningSum[i] = sum(nums[0..i]).

Вход: [1, 2, 3, 4]
Выход: [1, 3, 6, 10]
Вход: [1, 1, 1, 1, 1]
Выход: [1, 2, 3, 4, 5]

Суть задачи

Это буквально и есть построение префиксного массива! Только без нулевого элемента в начале и без запросов. Идеальная первая задача.

def runningSum(nums):
    result = []
    total = 0

    for num in nums:
        total += num
        result.append(total)

    return result

# Или ещё короче:
def runningSum(nums):
    for i in range(1, len(nums)):
        nums[i] += nums[i - 1]
    return nums
Время
O(n)
Память
O(n)
💡 Трассировка [1,2,3,4]:
i=0: total=1, result=[1]
i=1: total=3, result=[1,3]
i=2: total=6, result=[1,3,6]
i=3: total=10, result=[1,3,6,10] ✓

B. Экспедиция — разбор

Условие

N дней (2<=N<=100). В первый день приезжают A₀ школьников. Далее N-2 пар чисел (уезжают / приезжают). В последний день все уезжают. Найти максимальное количество человек одновременно в лагере.

Пример:
5 дней, 10 приехали
3 1 / 5 10 / 0 2
15 уехали
Ответ: 15

Алгоритм

1
Читаем первый день → добавляем приехавших
2
Для каждого промежуточного дня: people = people - уехало + приехало
3
На каждом шаге обновляем максимум
💡 people — это текущая накопительная сумма изменений. Это и есть идея prefix sum, применённая на ходу без отдельного массива. 
n = int(input())
people = 0
max_people = 0

# Первый день — только приезжают
people += int(input())
max_people = people

# Промежуточные дни
for _ in range(n - 2):
    leave, arrive = map(int, input().split())
    people = people - leave + arrive
    max_people = max(max_people, people)

# Последний день — все уезжают (не важен)
input()

print(max_people)
Время
O(n)
Память
O(1)

#303 · Easy

Range Sum Query — Immutable ↗

Условие

Реализовать класс NumArray. Метод sumRange(left, right) вызывается много раз. Нужно O(1) на запрос.

nums = [-2, 0, 3, -5, 2, -1]
sumRange(0, 2) → 1
sumRange(2, 5) → -1
sumRange(0, 5) → -3

Подсказки

💡 Подсказка 1: где строить prefix?
В конструкторе __init__ — один раз при создании объекта. Тогда каждый вызов sumRange будет O(1).
💡 Подсказка 2: какая формула?
prefix[right+1] - prefix[left] — используем массив со сдвигом (нулевой элемент = 0).
💡 Подсказка 3: проверь на примере
prefix = [0, -2, -2, 1, -4, -2, -3]
sumRange(0,2) = prefix[3]-prefix[0] = 1-0 = 1 ✓

🔑 Решение

Показать полное решение 
class NumArray:
    def __init__(self, nums):
        n = len(nums)
        self.prefix = [0] * (n + 1)
        for i in range(n):
            self.prefix[i+1] = self.prefix[i] + nums[i]

    def sumRange(self, left, right):
        return self.prefix[right+1] - self.prefix[left]

# Трассировка:
# nums   = [-2,  0,  3, -5,  2, -1]
# prefix = [ 0, -2, -2,  1, -4, -2, -3]
#
# sumRange(2,5) = prefix[6]-prefix[2]
#              = -3 - (-2) = -1 ✓
Построение
O(n)
Запрос
O(1)
✅ Это эталонная задача на prefix sum — если понял её, понял всё!

#724 · Easy

Find Pivot Index ↗

Условие

Найти индекс i, где сумма слева = сумма справа. Вернуть -1 если нет.

[1,7,3,6,5,6] → 3 (слева=11, справа=11)
[1,2,3] → -1
[2,1,-1] → 0 (слева=0, справа=0)

Подсказки

💡 Подсказка 1: как посчитать сумму справа?
Зная total = sum(nums) и left_sum, сумма справа = total - left_sum - nums[i]. Не нужен отдельный массив!
💡 Подсказка 2: когда обновлять left_sum?
После проверки, не до! Иначе left_sum будет включать текущий элемент.
💡 Подсказка 3: крайний случай i=0
При i=0 слева нет элементов → left_sum=0. Формула работает автоматически: right = total - 0 - nums[0].

🔑 Решение

Показать полное решение 
def pivotIndex(nums):
    total = sum(nums)
    left_sum = 0

    for i, num in enumerate(nums):
        # right = total - left_sum - nums[i]
        if left_sum == total - left_sum - num:
            return i
        left_sum += num  # обновляем ПОСЛЕ проверки

    return -1

#560 · Medium

Subarray Sum Equals K ↗

Условие

Найти количество непрерывных подмассивов с суммой равной k.

nums=[1,1,1], k=2 → 2
nums=[1,2,3], k=3 → 2 ([1,2] и [3])

Подсказки

💡 Подсказка 1: как записать условие через prefix?
Сумма подмассива [i..j] = prefix[j+1] - prefix[i]. Нам нужно это равнялось k, то есть prefix[i] == prefix[j+1] - k.
💡 Подсказка 2: как использовать hashmap?
Храним в словаре сколько раз встречалась каждая prefix-сумма. Для текущей суммы проверяем — сколько раз встречалась prefix - k.
💡 Подсказка 3: зачем seen[0] = 1?
Это «пустой префикс» (сумма 0 до начала массива). Нужен для случая когда подмассив начинается с индекса 0.

🔑 Решение

Показать полное решение 
from collections import defaultdict

def subarraySum(nums, k):
    count = 0
    prefix = 0
    seen = defaultdict(int)
    seen[0] = 1  # пустой префикс!

    for num in nums:
        prefix += num
        # ищем сколько раз была сумма prefix-k
        count += seen[prefix - k]
        seen[prefix] += 1

    return count

# Трассировка nums=[1,1,1], k=2:
# prefix=1: seen[1-2]=-1 → 0; seen={0:1,1:1}
# prefix=2: seen[2-2]=0 → +1; seen={0:1,1:1,2:1}
# prefix=3: seen[3-2]=1 → +1; count=2 ✓
Время
O(n)
Память
O(n)
⚠️ Это Medium — комбинация prefix + hashmap. Не расстраивайся если не с первого раза! Сначала закрой #303 и #724.
