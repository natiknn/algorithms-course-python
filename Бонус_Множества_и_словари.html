Множества и словари

set и dict — хэш-таблицы за O(1)
— самые мощные инструменты для оптимизации алгоритмов

set dict O(1) поиск Counter defaultdict Two Sum
Мотивация

Зачем нужны dict и set?

Представь: есть список из миллиона чисел. Нужно быстро проверять — есть ли число в списке?

❌ Список — O(n) поиск

nums = [1, 5, 3, 8, 2, ...]  # 1 000 000 чисел

# Каждая проверка — пробегаем весь список
if 999999 in nums:  # O(n) — медленно!
    print("есть")

# 10⁶ проверок × 10⁶ элементов = 10¹² операций 😱
При n=10⁶ каждый in сравнивает до 10⁶ элементов. Это катастрофически медленно!

✅ Множество/словарь — O(1) поиск

nums_set = set(nums)  # O(n) один раз

# Каждая проверка — мгновенно!
if 999999 in nums_set:  # O(1) — хэш!
    print("есть")

# 10⁶ проверок × O(1) = 10⁶ операций 🚀
Хэш-таблица вычисляет hash(x) и сразу знает, где искать элемент — независимо от размера коллекции!
💡 Ключевой сигнал для использования set/dict: если нужно «проверить принадлежность», «посчитать частоты», «дедупликация» — сразу думай о хэш-таблице!
Теория — set

Множество: set

# Создание
s = {1, 2, 3}
s = set([1, 2, 2, 3])  # {1, 2, 3} — дубли убраны
s = set()              # пустое множество (не {}!)

# Добавление и удаление
s.add(4)               # {1, 2, 3, 4}
s.remove(2)            # {1, 3, 4}  — ошибка если нет
s.discard(99)          # ничего, даже если нет 99
s.pop()                # удалить случайный элемент

# Проверка
3 in s                 # True — O(1)
len(s)                 # количество элементов
⚠️ remove(x) бросает KeyError если x нет.
discard(x) — безопасный вариант. 
# Операции над множествами
a = {1, 2, 3, 4}
b = {3, 4, 5, 6}

a | b   # объединение    → {1,2,3,4,5,6}
a & b   # пересечение    → {3, 4}
a - b   # разность       → {1, 2}
a ^ b   # симм. разность → {1,2,5,6}

# Проверка подмножества
{1,2}.issubset({1,2,3})    # True
{1,2,3}.issuperset({1,2})  # True
{1,2}.isdisjoint({3,4})    # True (нет общих)

# Преобразования
list(s)   # в список
sorted(s) # отсортированный список
✅ Типичный паттерн: дедупликация
unique = list(set(arr))
Теория — frozenset и Диаграмма Венна

frozenset и операции над множествами

frozenset — неизменяемое множество

# frozenset можно использовать как ключ словаря
fs = frozenset([1, 2, 3])

d = {}
d[frozenset([1,2])] = "пара"
d[frozenset([1,2])]   # → "пара"

# Зачем: когда нужно хранить множество
# как ключ в dict или положить в set
pairs = set()
pairs.add(frozenset([1, 2]))
pairs.add(frozenset([2, 1]))  # та же пара!
# pairs = {frozenset({1, 2})}  — дубль не добавился

Set comprehension

# Аналог list comprehension, но для set
squares = {x**2 for x in range(10)}
# {0, 1, 4, 9, 16, 25, 36, 49, 64, 81}

# Только чётные квадраты
even_sq = {x**2 for x in range(10) if x % 2 == 0}
# {0, 4, 16, 36, 64}

Теория — dict

Словарь: dict

# Создание
d = {"apple": 5, "banana": 3}
d = dict(apple=5, banana=3)
d = {}  # пустой словарь

# Чтение и запись
d["apple"]           # 5 — KeyError если нет
d.get("cherry")      # None — безопасно
d.get("cherry", 0)   # 0 — значение по умолчанию

d["cherry"] = 10     # добавить/обновить
del d["apple"]       # удалить ключ

# Проверка
"banana" in d        # True — O(1)
"grape" not in d     # True

# Обход
for key in d:                      # ключи
    print(key)
for key, val in d.items():         # пары
    print(key, val)
for val in d.values():             # значения
    print(val)
# Полезные методы
d.keys()        # dict_keys(['banana','cherry'])
d.values()      # dict_values([3, 10])
d.items()       # dict_items([('banana',3),...])

d.update({"grape": 7})  # добавить из другого dict
d.pop("banana")         # удалить и вернуть значение
d.setdefault("mango",0) # вернуть или установить дефолт

# Dict comprehension
squares = {x: x**2 for x in range(5)}
# {0:0, 1:1, 2:4, 3:9, 4:16}

# Частоты (классический паттерн)
freq = {}
for ch in "mississippi":
    freq[ch] = freq.get(ch, 0) + 1
# {'m':1, 'i':4, 's':4, 'p':2}
✅ Ключи словаря должны быть неизменяемыми: str, int, tuple, frozenset — да. list, dict, set — нет!
Теория — collections

collections: Counter, defaultdict, OrderedDict

Counter — счётчик частот

from collections import Counter

c = Counter("abracadabra")
# Counter({'a': 5, 'b': 2, 'r': 2, 'c': 1, 'd': 1})

c["a"]           # 5
c["z"]           # 0 (не KeyError!)
c.most_common(2) # [('a',5), ('b',2)]
c.total()        # 11 (сумма всех)

# Арифметика счётчиков
c1 = Counter("aab")
c2 = Counter("abc")
c1 + c2   # Counter({'a':3,'b':2,'c':1})
c1 - c2   # Counter({'a':1}) (только положит.)
c1 & c2   # Counter({'a':1,'b':1}) (минимум)

defaultdict — словарь с дефолтом

from collections import defaultdict

# Список по умолчанию
graph = defaultdict(list)
graph[1].append(2)  # не нужно проверять
graph[1].append(3)
# defaultdict(list, {1: [2, 3]})

# Счётчик по умолчанию
freq = defaultdict(int)
for ch in "hello":
    freq[ch] += 1
# defaultdict(int, {'h':1,'e':1,'l':2,'o':1})
💡 defaultdict(list) — идеален для построения графов и группировки.
defaultdict(int) — замена паттерну d.get(k,0)+1
Паттерны применения

Типовые паттерны с dict и set

1. Подсчёт частот

from collections import Counter

# Наиболее частые k элементов
def top_k(nums, k):
    return Counter(nums).most_common(k)

# Анаграммы — одинаковые ли символы?
def is_anagram(s, t):
    return Counter(s) == Counter(t)

2. Группировка

from collections import defaultdict

def group_anagrams(words):
    groups = defaultdict(list)
    for w in words:
        key = tuple(sorted(w))  # ключ = сорт. кортеж
        groups[key].append(w)
    return list(groups.values())

group_anagrams(["eat","tea","tan","ate","nat","bat"])
# [['eat','tea','ate'], ['tan','nat'], ['bat']]

3. Дедупликация с сохранением порядка

def unique_ordered(lst):
    seen = set()
    result = []
    for x in lst:
        if x not in seen:
            seen.add(x)
            result.append(x)
    return result

unique_ordered([3,1,4,1,5,9,2,6,5,3])
# [3, 1, 4, 5, 9, 2, 6]

4. Мемоизация (кэш результатов)

def fib(n, memo={}):
    if n in memo:
        return memo[n]
    if n <= 1:
        return n
    memo[n] = fib(n-1, memo) + fib(n-2, memo)
    return memo[n]

Two Sum — разбор через хэш-таблицу

Условие

Дан массив nums и число target. Найти индексы двух чисел, которые в сумме дают target.

Вход: nums=[2,7,11,15], target=9
Выход: [0,1] (2+7=9)

Алгоритм через dict

1
Храним в словаре: seen[число] = индекс
2
Для каждого nums[i] ищем target - nums[i] в словаре
3
Если нашли — вернуть [seen[complement], i]
def twoSum(nums, target):
    seen = {}  # число → индекс

    for i, num in enumerate(nums):
        complement = target - num

        if complement in seen:  # O(1)!
            return [seen[complement], i]

        seen[num] = i

    return []

# Трассировка: nums=[2,7,11,15], target=9
# i=0, num=2: complement=7, seen={}     → нет, seen={2:0}
# i=1, num=7: complement=2, seen={2:0}  → есть! → [0,1] ✓
Время
O(n)
Память
O(n)
✅ vs наивный O(n²) с двумя циклами — dict даёт O(n)!
💡 Этот паттерн повсюду: «найти два элемента с нужным свойством» → словарь для O(1) поиска.

Итог

Что запомнить

🔢 set

  • O(1) — add, remove, in
  • Уникальные элементы
  • | & - ^ — операции над множествами
  • frozenset — неизменяемый
  • Дедупликация: set(lst)

📖 dict

  • O(1) — get, set, in
  • d.get(k, default) — безопасно
  • d.items() — итерация пар
  • defaultdict — автоматический дефолт
  • Counter — подсчёт частот

🎯 Паттерны

  • Частоты → Counter
  • Группировка → defaultdict(list)
  • Дедупликация → set
  • Two Sum → dict дополнений
  • Кэш → dict результатов
🎯 На экзамене: если видишь «найти дубликаты», «подсчитать частоты», «проверить принадлежность», «найти пару с нужной суммой» — сразу используй set/dict!
