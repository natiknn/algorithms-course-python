Теория

Что такое жадный алгоритм?

Жадный алгоритм — это стратегия, при которой на каждом шаге мы выбираем наилучший вариант прямо сейчас, не думая о последствиях. Надеемся, что серия локально оптимальных решений приведёт к глобально оптимальному результату.

Аналогия из жизни

Представь, что ты выбираешь маршрут в горах. Жадная стратегия — всегда идти вверх по самому крутому склону. Иногда это приведёт на вершину, но иногда — на промежуточный холм.

✅ Жадный работает, когда локальный оптимум гарантированно ведёт к глобальному. Например: задача о монетах с номиналами 1, 5, 10, 25.
❌ Жадный не работает, когда нужно «пожертвовать» сейчас ради лучшего результата позже. Тогда нужно ДП.

Шаблон жадного алгоритма

1
Отсортировать данные по нужному критерию
2
Жадно выбирать лучший доступный вариант
3
Обновить состояние (уменьшить остаток, добавить в ответ)
4
Повторять пока задача не решена
💡 Ключевой вопрос: «Можно ли всегда брать лучшее прямо сейчас?»
Классический пример

Задача о монетах — объясняем жадность

Нужно набрать сумму 41 рубль монетами номиналов 25, 10, 5, 1. Используй как можно меньше монет.

Жадная стратегия

Всегда берём максимальный номинал, который не превышает остаток.

1
Остаток 41. Берём 25. Остаток: 16.
2
Остаток 16. Берём 10. Остаток: 6.
3
Остаток 6. Берём 5. Остаток: 1.
4
Остаток 1. Берём 1. Остаток: 0. Готово!
Итого: 4 монеты — это оптимум!
25 + 10 + 5 + 1 = 41 ✓

def min_coins(amount, coins):
    coins.sort(reverse=True)  # по убыванию
    count = 0
    result = []

    for coin in coins:
        while amount >= coin:
            amount -= coin
            result.append(coin)
            count += 1

    return count, result

# min_coins(41, [1, 5, 10, 25])
# → (4, [25, 10, 5, 1])
⚠️ Ловушка! С номиналами 1, 3, 4 и суммой 6 жадный даёт 4+1+1 = 3 монеты, хотя оптимум 3+3 = 2 монеты. Жадный не всегда работает!

Паттерн 1
Сортировка + жадный выбор

Паттерн 1: Сортируем и берём лучшее

Самый частый паттерн. Сначала сортируем элементы по нужному параметру, затем жадно выбираем по одному.

#455 Assign Cookies ↗

Детям нужны печенья. Ребёнок i доволен если размер печенья >= g[i]. Максимизировать количество довольных детей.

Идея: сортируем детей и печенья. Наименее жадному ребёнку даём наименьшее подходящее печенье.

g=[1,2,3], s=[1,1] → 1 ребёнок
g=[1,2], s=[1,2,3] → 2 ребёнка 
# #455 Assign Cookies
def findContentChildren(g, s):
    g.sort()   # дети по жадности
    s.sort()   # печенья по размеру
    child = 0
    cookie = 0

    while child < len(g) and cookie < len(s):
        if s[cookie] >= g[child]:
            child += 1   # ребёнок доволен
        cookie += 1      # печенье использовано

    return child
Время
O(n log n)
Память
O(1)
💡 Почему работает: если наименьший cookie не подходит наименее жадному ребёнку — он не подойдёт никому. Смело выбрасываем.

Паттерн 2
Локальный оптимум на массиве

Паттерн 2: Обновляем максимум/минимум на ходу

Идём по массиву и на каждом шаге жадно обновляем один параметр — лучшую цену, максимальный охват, текущую сумму.

#121 Best Time to Buy and Sell Stock ↗

Купить акцию в один день, продать в другой. Максимизировать прибыль.

Жадная идея: на каждый день запоминаем минимальную цену покупки слева и считаем прибыль если продаём сегодня.

prices=[7,1,5,3,6,4] → прибыль 5 (купить за 1, продать за 6)

#55 Jump Game ↗

Массив прыжков. Можно ли добраться до конца?

Идея: отслеживаем максимально достижимый индекс max_reach. Если текущий индекс > max_reach — застряли.

[2,3,1,1,4] → True (можно добраться)
[3,2,1,0,4] → False (застреваем)
# leetcode.com/problems/best-time-to-buy-and-sell-stock/
def maxProfit(prices):
    min_price = float('inf')
    max_profit = 0

    for price in prices:
        min_price = min(min_price, price)
        profit = price - min_price
        max_profit = max(max_profit, profit)

    return max_profit
# leetcode.com/problems/jump-game/
def canJump(nums):
    max_reach = 0

    for i in range(len(nums)):
        if i > max_reach:
            return False   # недостижимая позиция
        max_reach = max(max_reach, i + nums[i])

    return True
💡 Оба решения — один проход O(n). Ключ: один параметр, который обновляем жадно на каждом шаге.

Паттерн 3
Интервалы — сортировка по концу

Паттерн 3: Задачи на интервалы

Есть набор отрезков (start, end). Нужно выбрать максимальное количество непересекающихся или удалить минимальное для устранения пересечений.

Ключевая идея: сортируй по концу

Жадный выбор: всегда берём интервал с наименьшим концом — он освобождает место для максимального числа следующих.

1
Сортируем интервалы по времени окончания
2
Берём первый интервал, запоминаем его конец
3
Следующий берём только если start >= предыдущий end
Пример: митинги [9-10], [9-12], [10-11], [11-12]
Сортируем по концу: [9-10], [10-11], [11-12], [9-12]
Берём [9-10], [10-11], [11-12] → 3 митинга

#435 Non-overlapping Intervals ↗

Удалить минимальное количество интервалов, чтобы не было пересечений.

def eraseOverlapIntervals(intervals):
    if not intervals:
        return 0

    # сортируем по концу
    intervals.sort(key=lambda x: x[1])

    count = 0          # удалённых
    end = intervals[0][1]  # конец последнего взятого

    for i in range(1, len(intervals)):
        if intervals[i][0] < end:
            # пересекается → удаляем текущий
            count += 1
        else:
            # не пересекается → берём
            end = intervals[i][1]

    return count
Время
O(n log n)
Память
O(1)

Теория

Когда жадный работает, а когда нет?

✅ Жадный работает

  • Задача о монетах с «хорошими» номиналами (1, 5, 10, 25)
  • Assign Cookies — раздача печенья
  • Jump Game — можно ли добраться
  • Выбор непересекающихся интервалов
  • Best Time to Buy and Sell Stock
  • «Завоевание»
Признак: локальный оптимум не закрывает путь к глобальному. Можно доказать, что жадный выбор не хуже любого другого.

❌ Жадный не работает

  • Задача о монетах с произвольными номиналами (1, 3, 4)
  • Задача о рюкзаке (0/1 Knapsack)
  • Поиск длиннейшей общей подпоследовательности (НОП)
  • Задача коммивояжёра
Признак: нужно «пожертвовать» сейчас ради лучшего результата потом. Локальный оптимум != глобальный.
💡 Эмпирическое правило: если в задаче есть слова «максимум», «минимум», «оптимально» — попробуй жадный. Если не работает — ДП.
