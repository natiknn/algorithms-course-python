Теория

Что такое динамическое программирование?

ДП — это стратегия решения задач, у которых есть перекрывающиеся подзадачи. Вместо того чтобы решать одно и то же снова и снова — запоминаем результат.

💡 Аналогия: ты уже считал сколько маршрутов из A в B. Когда снова понадобится — просто смотришь в записную книжку, не считаешь заново.
1
Оптимальная подструктура — решение задачи строится из решений подзадач
2
Перекрывающиеся подзадачи — одни и те же подзадачи встречаются много раз
3
Мемоизация / табуляция — сохраняем результаты, не пересчитываем

❌ Без ДП — экспоненциально

fib(5) вызывает fib(3) дважды, fib(2) трижды...

def fib(n):
    if n <= 1: return n
    return fib(n-1) + fib(n-2)
# O(2ⁿ) — катастрофически медленно!

✅ С ДП — линейно

memo = {}
def fib(n):
    if n <= 1: return n
    if n in memo: return memo[n]
    memo[n] = fib(n-1) + fib(n-2)
    return memo[n]
# O(n) — быстро!
Теория — Подходы

Два подхода: мемоизация и табуляция

📝 Мемоизация (top-down)

Рекурсия + кэш. Считаем «сверху вниз»: от большой задачи к маленьким. Запоминаем уже вычисленное.

memo = {}

def dp(n):
    if n <= 1:
        return n          # базовый случай
    if n in memo:
        return memo[n]    # уже считали!
    memo[n] = dp(n-1) + dp(n-2)
    return memo[n]
  • ✅ Проще написать
  • ✅ Считает только нужные подзадачи
  • ⚠️ Рекурсия — лимит стека

📊 Табуляция (bottom-up)

Итерация + массив. Считаем «снизу вверх»: от маленьких задач к большим. Заполняем таблицу.

def dp(n):
    if n <= 1:
        return n
    table = [0] * (n + 1)
    table[1] = 1

    for i in range(2, n + 1):
        table[i] = table[i-1] + table[i-2]

    return table[n]
  • ✅ Нет рекурсии — нет лимита стека
  • ✅ Чуть быстрее на практике
  • ⚠️ Нужно сразу придумать порядок заполнения
🎯 Что выбрать? Для начала — мемоизацию. Когда поймёшь задачу — перепиши через табуляцию. Оба подхода засчитываются.

🪜 Задача о лестнице — первое ДП

Условие

Лестница из N ступенек. За один шаг можно подняться на 1 или 2 ступеньки. Сколько различных способов добраться до вершины?

1
2
3
4
5 🏁
N=3 → 3 способа: (1+1+1), (1+2), (2+1)
N=4 → 5 способов

Ключевое наблюдение

На ступеньку N можно попасть только с N-1 или N-2. Значит: dp[n] = dp[n-1] + dp[n-2]

Это Фибоначчи! Вот табуляция:

def climbStairs(n):
    if n <= 2:
        return n

    dp = [0] * (n + 1)
    dp[1] = 1   # 1 способ добраться до ступеньки 1
    dp[2] = 2   # 2 способа до ступеньки 2

    for i in range(3, n + 1):
        dp[i] = dp[i-1] + dp[i-2]

    return dp[n]

dp[5] = dp[4]+dp[3] = 5+3 = 8 способов ✓

House Robber — второй шаг ↗

Условие

Ряд домов, в каждом сумма денег. Нельзя грабить два соседних дома — сработает сигнализация. Найти максимальную сумму.

nums=[1,2,3,1] → 4 (берём дома 0 и 2)
nums=[2,7,9,3,1] → 12 (берём дома 0, 2, 4)

Формула ДП

На каждом доме выбираем: ограбить (взять nums[i] + dp[i-2]) или пропустить (взять dp[i-1]).

dp[i] = max(dp[i-1], nums[i] + dp[i-2])
⚠️ Связь с «Удали и заработай» ! Там ровно та же идея — берёшь x, теряешь x±1. 
def rob(nums):
    n = len(nums)
    if n == 1: return nums[0]

    dp = [0] * n
    dp[0] = nums[0]
    dp[1] = max(nums[0], nums[1])

    for i in range(2, n):
        # взять i-й + лучшее до i-2
        # или не брать i-й — лучшее до i-1
        dp[i] = max(dp[i-1],
                    nums[i] + dp[i-2])

    return dp[n-1]

41111+11=1212 ✓

Coin Change ↗

Условие

Даны монеты разных номиналов и сумма amount. Найти минимальное количество монет для набора суммы. Монет каждого номинала неограниченно.

coins=[1,5,11], amount=15 → 3 (5+5+5)
coins=[1,2,5], amount=11 → 3 (5+5+1)
coins=[2], amount=3 → -1 (невозможно)

Формула ДП

dp[x] = минимальное число монет для суммы x.

dp[x] = min(dp[x-coin]+1) для всех coin
💡 Связь с «Завоеванием»: там тоже dp[x] = минимальная стоимость иметь x воинов! 
def coinChange(coins, amount):
    INF = float('inf')
    dp = [INF] * (amount + 1)
    dp[0] = 0  # сумму 0 набрать — 0 монет

    for x in range(1, amount + 1):
        for coin in coins:
            if x >= coin:
                dp[x] = min(dp[x],
                            dp[x - coin] + 1)

    return dp[amount] if dp[amount] != INF else -1

B. Удали и заработай — разбор

Условие

Массив натуральных чисел nums (1<=n<=2×10⁴, 1<=nums[i]<=10⁴). Операция: выбрать nums[i], получить nums[i] бурлей, удалить все элементы равные nums[i]±1. Максимизировать сумму.

[1,2,3] → 4 (берём 1 и 3, удаляется 2)
[2,2,3,3,4] → 9 (берём оба 3, удаляются 2 и 4)

Сведение к House Robber

1
Создаём массив gain[v] = сумма всех элементов равных v
2
Если берём значение v — нельзя брать v-1 и v+1
3
Это ровно House Robber на массиве gain!
def deleteAndEarn(nums):
    max_val = max(nums)
    # gain[v] = сколько заработаем взяв все v
    gain = [0] * (max_val + 1)
    for num in nums:
        gain[num] += num

    # Теперь House Robber на gain[]
    if max_val == 0:
        return 0

    dp = [0] * (max_val + 1)
    dp[1] = gain[1]

    for i in range(2, max_val + 1):
        dp[i] = max(dp[i-1],
                    gain[i] + dp[i-2])

    return dp[max_val]

Пример [1, 2, 3] → ответ 4:

gain = [0, 1, 2, 3]
dp[1] = gain[1] = 1
dp[2] = max(dp[1], gain[2]+dp[0]) = max(1, 2+0) = 2
dp[3] = max(dp[2], gain[3]+dp[1]) = max(2, 3+1) = 4 ✓

Берём значения 1 и 3 (нельзя брать 2): 1 + 3 = 4
Время
O(n+k)
Память
O(k)

B. Удали и заработай — что хранят gain[] и dp[]?

Массив gain[]

gain = [0] * (max_val + 1)
for num in nums:
    gain[num] += num

Индекс — само значение числа.
Значение — сколько очков получишь, если возьмёшь все числа, равные этому индексу.

Пример: nums = [2, 2, 3, 3, 4]
индекс01234
gain00464
gain[2] = 4, потому что две двойки: 2+2=4
gain[3] = 6, потому что две тройки: 3+3=6
gain[4] = 4, одна четвёрка: 4
⚡ Теперь задача — House Robber на gain[]: нельзя брать соседние ячейки!

Массив dp[]

dp = [0] * (max_val + 1)
dp[1] = gain[1]

for i in range(2, max_val + 1):
    dp[i] = max(dp[i-1],
                gain[i] + dp[i-2])

Индекс i — рассматриваем числа из диапазона [1..i].
Значение — максимум очков, достижимый используя только числа <= i.

dp[i] = max(dp[i-1], gain[i] + dp[i-2])
          ↑ не берём i   ↑ берём i, теряем i-1

Почему длина max_val + 1?

Оба массива индексируются по значению числа. Значения идут от 0 до max_val включительно — нужно max_val + 1 ячеек.

Если max_val = 4 → нужны индексы 0, 1, 2, 3, 4 → это 5 ячеек = 4 + 1

НОП — Наибольшая Общая Подпоследовательность

Задача (глава 9 книги)

Даны две строки. Найти длину их наибольшей общей подпоследовательности — символы не обязательно стоят рядом, но порядок сохраняется.

«FISH» и «FOSH» → 3 (FSH)
«HISH» и «VISTA» → 2 (IS)
⚠️ Это не подстрока! Символы могут быть не рядом.

Почему здесь нужен 2D-массив?

У нас два изменяемых параметра: позиция в первой строке и позиция во второй. Одномерного массива не хватит — нужна таблица.

dp[i][j] = длина НОП для s1[:i] и s2[:j]

Правило заполнения

✓
Если s1[i-1] == s2[j-1] → dp[i][j] = dp[i-1][j-1] + 1
✗
Иначе → dp[i][j] = max(dp[i-1][j], dp[i][j-1])

НОП — код и пошаговая трассировка

Реализация

def lcs(s1, s2):
    n, m = len(s1), len(s2)
    # Создаём таблицу (n+1) × (m+1), заполненную нулями
    dp = [[0] * (m + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        for j in range(1, m + 1):
            if s1[i-1] == s2[j-1]:
                # Символы совпали — берём диагональ + 1
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                # Берём максимум из левой или верхней ячейки
                dp[i][j] = max(dp[i-1][j],
                               dp[i][j-1])

    return dp[n][m]

print(lcs("FISH", "FOSH"))  # → 3
Строк в таблице
n + 1
Столбцов
m + 1

Почему +1 в размере таблицы?

Строка с индексом 0 и столбец с индексом 0 — это базовый случай: пустая строка. НОП пустой строки с любой другой = 0.

dp[0][j] = 0 для всех j — пустая s1
dp[i][0] = 0 для всех i — пустая s2

Как читать таблицу при совпадении?

↖
←
↑
?
Если символы совпали →
dp[i][j] = dp[i-1][j-1] + 1

Иначе →
dp[i][j] = max(↑, ←)
✅ Итог: 2D ДП = те же идеи что и 1D, но состояние описывается двумя параметрами. Таблица вместо массива — всё остальное одинаково.

E. Упаковка символов — разбор

Условие

Упаковать строку из A-Z (длина <=100) в минимальное представление. Правила: X(S) = S повторяется X раз. Вложенность допускается.

AAAAAAAABABABCCD → 9(A)3(AB)CCD
NEERCYESYESNEERCYESYESYES → 2(NEERC3(YES))

Идея ДП по подстрокам

dp[i][j] = минимальная длина упаковки подстроки s[i..j].

1
Если подстрока — это k повторений T, то кодируем как k(dp[T])
2
Иначе делим на две части и берём сумму: dp[i][m] + dp[m+1][j]
3
Берём минимум из всех вариантов
def encode(s):
    n = len(s)
    # dp[i][j] = минимальная упаковка s[i..j]
    dp = [[''] * n for _ in range(n)]

    for length in range(1, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            sub = s[i:j+1]
            dp[i][j] = sub  # без упаковки

            # Проверяем повторения
            t = (sub + sub).find(sub, 1)
            if t != len(sub):
                k = len(sub) // t
                rep = dp[i][i + t - 1]
                cand = str(k) + '(' + rep + ')'
                if len(cand) < len(dp[i][j]):
                    dp[i][j] = cand

            # Делим на две части
            for m in range(i, j):
                cand = dp[i][m] + dp[m+1][j]
                if len(cand) < len(dp[i][j]):
                    dp[i][j] = cand

    return dp[0][n-1]
Время
O(n⁴)
Память
O(n²)

D. Удали строку — разбор

Условие

За операцию: удалить одинаковый непустой префикс и суффикс из одних и тех же букв. Найти минимальную длину.

abba → 0 (a+a → bb → b+b → "")
aboba → 1 (a+a → bob → b+b → "o")
zzzaabxxxcazz → 5

Почему нужно ДП?

Жадный (два указателя) не работает на строке типа zzzaabxxxcazz: удаляем z, остаётся aabxxxca, и теперь a+a = убираем, остаётся bxxxc = 5. Жадный дал бы неверный ответ на некоторых входах.

⚠️ Это задача уровня D. Полное ДП-решение требует dp[i][j] = минимальная длина подстроки s[i..j].
def minLength(s):
    n = len(s)
    # dp[i][j] = мин длина s[i..j] после операций
    dp = [[0]*n for _ in range(n)]

    # Базовый: одиночные символы
    for i in range(n):
        dp[i][i] = 1

    # Заполняем по возрастанию длины
    for length in range(2, n + 1):
        for i in range(n - length + 1):
            j = i + length - 1
            dp[i][j] = length  # без операций

            # Если крайние символы одинаковы
            if s[i] == s[j]:
                # Удаляем крайние блоки
                li, rj = i, j
                c = s[i]
                while li <= rj and s[li] == c:
                    li += 1
                while li <= rj and s[rj] == c:
                    rj -= 1
                if li > rj:
                    dp[i][j] = 0
                else:
                    dp[i][j] = dp[li][rj]

            # Делим на части
            for m in range(i, j):
                dp[i][j] = min(dp[i][j],
                    dp[i][m] + dp[m+1][j])

    return dp[0][n-1]
Время
O(n³)
Память
O(n²)

Delete and Earn ↗

Условие

Берёшь nums[i] → получаешь nums[i] очков → удаляются все nums[i]-1 и nums[i]+1. Максимизировать очки.

nums=[3,4,2] → 6 (берём 3 и теряем 2 и 4)
nums=[2,2,3,3,4] → 9

Подсказки

💡 Подсказка 1: как преобразовать?
Создай массив gain[v] = сумма всех элементов равных v. Тогда взять v = получить gain[v] и нельзя брать v±1.
💡 Подсказка 2: какая задача получилась?
House Robber! Нельзя брать соседние значения. Применяем dp[i] = max(dp[i-1], gain[i]+dp[i-2]).
💡 Подсказка 3: базовый случай?
dp[0]=0, dp[1]=gain[1]. Если max_val=1 — сразу return gain[1].

🔑 Решение

Показать полное решение 
def deleteAndEarn(nums):
    if not nums: return 0
    max_val = max(nums)
    gain = [0] * (max_val + 1)
    for num in nums:
        gain[num] += num

    if max_val == 1:
        return gain[1]

    dp = [0] * (max_val + 1)
    dp[1] = gain[1]
    for i in range(2, max_val + 1):
        dp[i] = max(dp[i-1],
                    gain[i] + dp[i-2])
    return dp[max_val]

# Трассировка [2,2,3,3,4]:
# gain=[0,0,4,6,4]
# dp[1]=0, dp[2]=4, dp[3]=max(4,6+0)=6
# dp[4]=max(6,4+4)=8... но LeetCode
# даёт 9 для [2,2,3,3,4]?
# Проверим: 3+3=6 (берём тройки), итого 6
# Или 2+2=4 + 4=4 (берём двойки и четвёрки)=8
# Ответ: 9 для [3,4,2]→ gain[2]=2,gain[3]=3,gain[4]=4
# dp[2]=2, dp[3]=max(2,3+0)=3, dp[4]=max(3,4+2)=6 ✓
Время
O(n+k)
Память
O(k)

Coin Change ↗

Условие

Монеты разных номиналов, сумма amount. Найти минимальное количество монет. Монет каждого вида — сколько угодно.

coins=[1,2,5], amount=11 → 3 (5+5+1)
coins=[2], amount=3 → -1

Подсказки

💡 Подсказка 1: что хранит dp[x]?
dp[x] = минимальное количество монет для суммы x. dp[0]=0 (сумму 0 набрать — 0 монет).
💡 Подсказка 2: формула перехода?
Для каждой монеты c: если x >= c, то dp[x] = min(dp[x], dp[x-c]+1). Перебираем все монеты.
💡 Подсказка 3: что вернуть?
dp[amount]. Если осталось INF — вернуть -1 (сумма недостижима).

🔑 Решение

Показать полное решение 
def coinChange(coins, amount):
    INF = float('inf')
    dp = [INF] * (amount + 1)
    dp[0] = 0

    for x in range(1, amount + 1):
        for coin in coins:
            if x >= coin and dp[x-coin] != INF:
                dp[x] = min(dp[x],
                            dp[x - coin] + 1)

    return -1 if dp[amount] == INF else dp[amount]

# Трассировка coins=[1,2,5], amount=6:
# dp = [0, 1, 1, 2, 2, 1, 2]
# dp[6] = min(dp[5]+1, dp[4]+1, dp[1]+1)
#       = min(2, 3, 2) = 2 ✓ (5+1)
Время
O(amount×coins)
Память
O(amount)
💜 Связь: «Завоевание» — ровно такая же структура: dp[x] = минимальная стоимость иметь x воинов. Переход: для каждого города пробуем купить k воинов.

Decode String ↗

Условие

Раскодировать строку вида k[encoded] — encoded повторяется k раз. Вложенность допустима.

"3[a]2[bc]" → "aaabcbc"
"3[a2[c]]" → "accaccacc"
"2[abc]3[cd]ef" → "abcabccdcdcdef"

Это обратная задача к «Упаковке символов» ! Там мы кодировали, здесь — декодируем.

Подсказки

💡 Подсказка 1: какую структуру использовать?
Стек! Когда встречаем [ — кладём текущую строку и число на стек. Когда ] — достаём и повторяем.
💡 Подсказка 2: как обрабатывать вложенность?
На стеке храним пары (текущая строка до скобки, k). При закрытии скобки берём содержимое × k и добавляем к строке со стека.

🔑 Решение (через стек)

Показать полное решение 
def decodeString(s):
    stack = []
    cur_str = ""
    cur_num = 0

    for ch in s:
        if ch.isdigit():
            cur_num = cur_num * 10 + int(ch)
        elif ch == '[':
            # сохраняем состояние
            stack.append((cur_str, cur_num))
            cur_str = ""
            cur_num = 0
        elif ch == ']':
            # восстанавливаем и повторяем
            prev_str, k = stack.pop()
            cur_str = prev_str + cur_str * k
        else:
            cur_str += ch

    return cur_str

# Трассировка "3[a2[c]]":
# '3' → num=3
# '[' → stack=[("",3)], cur=""
# 'a' → cur="a"
# '2' → num=2
# '[' → stack=[("",3),("a",2)], cur=""
# 'c' → cur="c"
# ']' → pop("a",2): cur="a"+"c"*2="acc"
# ']' → pop("",3): cur=""+"acc"*3="accaccacc" ✓
Время
O(n·k)
Память
O(n)
